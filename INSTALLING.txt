## INSTALLING ##

### FROM BINARY ###

If you received your copy of Webfort as a Windows binary, this is the procedure
to follow.

1. Installing the plugin.

Web Fortress requires:
* Dwarf Fortress 53.x (the Steam release, v50+) or from the website
* A matching version of DFHack for DF 53.x
	https://github.com/dfhack/dfhack/releases

IMPORTANT: Web Fortress currently only supports ASCII graphics mode. Before
enabling the plugin, make sure Dwarf Fortress is running with the classic ASCII
renderer. Sprite graphics modes are not yet supported.

Extract this zip file over your Dwarf Fortress install directory.

To start Web Fortress, launch Dwarf Fortress and run the following command in
the DFHack console:

	enable webfort

You should see a line that says:

	Web Fortress started on port 1234

If you see that, Web Fortress has been properly installed.

2. Running the frontend

Web Fortress includes its own built-in HTTP server. Once the plugin is running,
open a browser and navigate to:

	http://localhost:1234/webfort.html

To allow remote players to connect, replace `localhost` with your public IP
address. Your IP can be found by typing "What is my IP Address" into Google.

3. Port Forwarding

For external users to connect to your webfort, they will need access to port
1234. This is typically done via port forwarding. If you haven't done this
before, see

	http://portforward.com/

for a list of guides for common routers.

### From Source ###

See COMPILING.md for build instructions. After building and installing via
the normal DFHack install process, run `enable webfort` in the DFHack console
to start the server. The rest of the steps above (accessing the frontend,
port forwarding) are the same.
